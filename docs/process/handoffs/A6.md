# A6 — Teams (Handoff, Phase 1)

- **Owning agent:** A6 · **Owns:** `backend/src/modules/teams/**`
- **Charter:** `docs/agents/build/A6-teams.md` · **Scope:** REQUIREMENTS §4
- **Related:** ADR-0002 (DB client), ADR-0003 (errors), ADR-0004 (auth), ADR-0005 (module registration), ADR-0006 (`@app/shared`); handoffs A1 (contract), A3 (persistence), A4 (backend core)

## What shipped

`backend/src/modules/teams/index.ts` — default-export `teamsModule: FastifyPluginAsync` (ADR-0005), to be mounted by A4 at prefix **`/api/teams`**. Five routes, each validated/serialized by `schemas[operationId]` from `@app/shared` (ADR-0006):

| Method + path (under prefix) | operationId | Behaviour |
|---|---|---|
| `GET /` | `listTeams` | 200; all teams ordered by `name` asc (contract). |
| `POST /` | `createTeam` | 201 with the created team; name trimmed; 409 `TEAM_NAME_TAKEN` on duplicate (any case); 400 `VALIDATION_ERROR` if empty after trim. |
| `GET /:teamId` | `getTeam` | 200; 404 `TEAM_NOT_FOUND` if absent. |
| `GET /:teamId/epics` | `listTeamEpics` | 200; team-scoped epic list ordered by `title` asc (ticket epic-dropdown source); 404 `TEAM_NOT_FOUND` if the team is absent. |
| `PATCH /:teamId` | `updateTeam` | 200 (rename); name trimmed; 404 `TEAM_NOT_FOUND`; 409 `TEAM_NAME_TAKEN`. |
| `DELETE /:teamId` | `deleteTeam` | 204 (empty body); 404 `TEAM_NOT_FOUND`; **409 `TEAM_HAS_DEPENDENTS`** if the team still has tickets OR epics (no cascade). |

### Contract adherence
- **DB access** exclusively via the frozen singleton `import { prisma } from 'backend/src/db'` (ADR-0002). No `PrismaClient` instantiated.
- **Errors** thrown as `ConflictError` / `NotFoundError` / `ValidationError` from `backend/src/core/errors` (ADR-0003); status is never set manually. `TEAM_HAS_DEPENDENTS` is a 409 exactly as required.
- **Auth** is global (ADR-0004); routes do not attach auth. All verified users manage all teams (no ownership/membership — §4). `request.authUser` is available but not needed for team ops.
- **Timestamps** `createdAt` / `modifiedAt` are server-set UTC by the DB (`@default(now())` / `@updatedAt` in A3's schema). The API never accepts them: create/update handlers write only `{ name }`, and the create/update body schemas are `additionalProperties: false`.

### Name rules (§4)
- Trimmed at the service layer, then rejected if empty (`" "` passes schema `minLength: 1` but is empty after trim → 400 `VALIDATION_ERROR`).
- Case-insensitive uniqueness is provided by the DB `citext` unique index on `teams.name` (A3 handoff). The service does not lowercase; it relies on the DB. A duplicate surfaces as Prisma `P2002` → mapped to 409 `TEAM_NAME_TAKEN`.

### Delete semantics (no cascade)
Delete first loads the team (404 if missing), then counts dependent tickets and epics in parallel; if either is > 0 it throws 409 `TEAM_HAS_DEPENDENTS`. As a race safety-net, the `prisma.team.delete` is wrapped: FK `RESTRICT` violations (`P2003`/`P2014`) also map to 409, and `P2025` to 404. This matches A3's FK actions (`Epic.team` / `Ticket.team` = RESTRICT).

## `listTeamEpics` — GATE-1 decision: OWNED by A6 (implemented)
`GET /teams/{teamId}/epics` (operationId `listTeamEpics`) lives under the `/api/teams` prefix this plugin owns, so per the coordinator's GATE-1 decision it is A6's. Implemented here as a **team-scoped epic READ**: it verifies the team exists (404 `TEAM_NOT_FOUND` otherwise), then returns `prisma.epic.findMany({ where: { teamId }, orderBy: { title: 'asc' } })` mapped to the contract's `Epic` shape (`id, teamId, title, description, createdAt, modifiedAt`; timestamps ISO-8601 UTC). It queries `prisma.epic` directly (DB boundary, ADR-0002) and does **not** import A15's epics module. Auth is global.

## Verification performed
- **`npm run build -w backend`** (`tsc && tsc-alias`) — **clean**.
- **Runtime behaviour** driven through a real `Fastify` instance with `registerErrorHandler` + `teamsModule` registered at `/api/teams`, using `fastify.inject` (this environment has no Postgres, so the shared `prisma` singleton's methods were stubbed on the same object the module imports). 16/16 acceptance-relevant checks pass:
  - list ordered array + ISO-8601 UTC timestamps; DTO shape correct.
  - create 201; name trimmed (`"  Gamma  "` → `Gamma`); whitespace-only → 400 `VALIDATION_ERROR`; missing name → 400.
  - **duplicate name any-case → 409 `TEAM_NAME_TAKEN`** (P2002 mapping).
  - get 200 / 404 `TEAM_NOT_FOUND`.
  - rename 200 (trimmed); duplicate → 409; missing → 404.
  - **delete with tickets → 409 `TEAM_HAS_DEPENDENTS`; delete with epics → 409**; missing → 404; **clean delete → 204 empty body**.
  - **timestamps server-set:** a client-supplied `createdAt` never reaches the write — the returned timestamp was the DB value, not the client's.
- **`listTeamEpics`** (added post-GATE-1) driven through the same inject harness (stubbed `prisma`): 3/3 pass — 200 team-scoped array ordered by title with correct `Epic` DTO shape; 404 `TEAM_NOT_FOUND` for a missing team.
