# A5 — Auth & Identity — Handoff

- **Owning agent:** A5 · **Owns:** `backend/src/modules/auth/**`, `backend/src/mail/**`
- **Source of truth:** REQUIREMENTS §3, `contracts/openapi.yaml` (auth ops)
- **Contracts consumed (unchanged):** ADR-0002 (DB client), ADR-0003 (errors), ADR-0004 (auth helpers / PUBLIC_ROUTES), ADR-0005 (module registration), ADR-0006 (`@app/shared` schemas), ADR-0007 (env)

## What shipped

### `backend/src/mail/**`
| File | Purpose |
|---|---|
| `mailer.ts` | Lazily-created, reused nodemailer transport built from the frozen `SMTP_*` env (via `getConfig`). `secure` only on port 465; auth attached only when `SMTP_USER` is set → works for dev Mailhog (1025, no auth) and prod `relay1.dataart.com` (587). |
| `verification-email.ts` | `buildVerificationEmail(verifyUrl)` → `{ subject, text, html }`. The ONLY link in the message is the single-use verification URL. |
| `index.ts` | `sendVerificationEmail(to, verifyUrl)` — composes + sends; throws on SMTP failure (caller decides). |

### `backend/src/modules/auth/**`
| File | Purpose |
|---|---|
| `password.ts` | `hashPassword` / `verifyPassword` using Argon2id (`@node-rs/argon2`). Plaintext never stored/logged. |
| `tokens.ts` | `issueVerificationToken(userId)` (256-bit random, +24h expiry) and `invalidateUnusedTokens(userId)` (deletes prior unused rows). |
| `service.ts` | `signup`, `verifyEmail`, `resendVerification`, `login` — the account lifecycle. |
| `index.ts` | **`export default authModule: FastifyPluginAsync`** (ADR-0005). Routes only; per-route `schema: schemas[operationId]` (ADR-0006). |

## Endpoint behavior (mounted at prefix `/api/auth` by A4)

| Op | Method + path | Behavior |
|---|---|---|
| `signup` | POST `/signup` | Trim+lowercase email; Argon2id hash; create unverified user; issue token + send verification email. **201** `{message}`. **409 EMAIL_TAKEN** on duplicate (DB unique index, race-safe via P2002). No auto-login. |
| `login` | POST `/login` | **200** `{token,user}` (bearer via `issueToken`). **401 INVALID_CREDENTIALS** for unknown email OR wrong password (indistinguishable). **403 EMAIL_NOT_VERIFIED** for valid-but-unverified. |
| `logout` | POST `/logout` | **204**. Authenticated (NOT in PUBLIC_ROUTES → global auth enforces). Stateless JWT: client discards token. |
| `verifyEmail` | GET `/verify-email?token=` | Consumes single-use token; atomically marks used + user verified. **302** → `${APP_BASE_URL}/login` (no auto-login). **400 TOKEN_INVALID / TOKEN_USED / TOKEN_EXPIRED**. |
| `resendVerification` | POST `/resend-verification` | Invalidates prior unused tokens, issues+sends a new one. **Always 202** regardless of account existence/state (no enumeration). |

## Key design notes
- **Email normalization:** trimmed + lowercased on write and lookup. DB uniqueness is enforced case-insensitively by the `citext` column (A3); the P2002 catch is the race-safe backstop.
- **Single-use + race-safe verify:** consumption is an `updateMany({ where: { id, usedAt: null } })` guard inside a transaction with the user update — a concurrent second verify claims 0 rows → `TOKEN_USED`.
- **Resend invalidation:** `deleteMany({ userId, usedAt: null })` before issuing the new token, so any prior link resolves as `TOKEN_INVALID`.
- **No token in URL except the verification token:** bearer JWTs travel only in the `Authorization` header (ADR-0004). The verification link is the sole URL-borne token (permitted by REQUIREMENTS §9).
- **SMTP resilience:** in `signup`/`resendVerification` a transient send failure is logged (`request.log`) but does NOT fail the request — the account exists and the user can resend. `sendVerificationEmail` itself still throws so this policy is explicit at the call site.
- **Unverified accounts blocked from the app:** enforced at `login` (403). All business endpoints already require a bearer token (global auth, ADR-0004), and a token is only issued to verified users — so unverified accounts cannot reach business endpoints.
- **Argon2id note:** `@node-rs/argon2` declares `Algorithm` as an ambient `const enum` (unusable by value under `isolatedModules`), so the value `2` (=Argon2id, also the library default) is pinned via a type-only import. Documented inline in `password.ts`.

## Verification performed
- **`npm run build -w backend`** (`tsc && tsc-alias`) — **clean**. tsc-alias correctly rewrote the frozen bare specifiers in emitted output (`backend/src/db` → `../../db/index.js`, `backend/src/core/*`, `backend/src/mail`).
- **Argon2id / no-plaintext (runtime, compiled output):** hash starts `$argon2id$`; never contains the plaintext; correct password verifies `true`, wrong `false`; two hashes of the same password differ (random salt).
- **Verification link (runtime):** every URL occurrence in the email (text + html) is the `…/api/auth/verify-email?token=…` link; no bearer/JWT reference present.
- **Plugin integration (runtime, real Fastify + A4 core, `fastify.inject`, no DB):**
  - All 5 routes registered under `/api/auth` (`printRoutes` tree confirms signup / login / logout / verify-email / resend-verification).
  - `signup` <8-char password → **400 VALIDATION_ERROR**; missing field → 400 (schema from `@app/shared`).
  - `login` empty body → 400; `verify-email` missing token → 400 (querystring schema).
  - `logout` without bearer → **401 UNAUTHORIZED** (global auth applies — logout is not public).
  - Valid-shaped public `signup` passes auth+validation and reaches the handler (then 500 only because Postgres is unreachable in this environment).
- **DB-dependent happy path** (signup→verify→login; expired/reused rejected; resend invalidates prior) is implemented and reasoned per the branch logic above; end-to-end exercise requires the compose Postgres (this environment has none, consistent with A3/A4 handoffs). First live run is on `docker compose up`.
