# A15 — Epics Backend (handoff)

**Scope:** Epic CRUD + team scoping (REQUIREMENTS §5). Closes audit C1 (previously-orphaned epics module).
**Files owned / written:** `backend/src/modules/epics/index.ts` (only). No other files touched.

## What was delivered
`epicsModule: FastifyPluginAsync` (default export) at `backend/src/modules/epics/index.ts`, mounted by A4 at
prefix `/api/epics` (ADR-0005). All routes attach their JSON Schema from `@app/shared` `schemas[operationId]`
(ADR-0006); DB access is the frozen singleton `import { prisma } from 'backend/src/db'` (ADR-0002); domain
outcomes are thrown as `DomainError` subclasses from `backend/src/core/errors` (ADR-0003); auth is global
(ADR-0004) and read-only here.

| Method | Path (after prefix) | operationId | Behaviour |
|---|---|---|---|
| GET | `/api/epics` (`?teamId=X` optional) | `listEpics` | All epics or team-scoped, ordered by title. **This is the team-scoped listing for the ticket epic dropdown.** |
| POST | `/api/epics` | `createEpic` | 201; validates team exists (→ 404 `TEAM_NOT_FOUND`); title trimmed + non-empty; team fixed at creation. |
| GET | `/api/epics/:epicId` | `getEpic` | 200 or 404 `EPIC_NOT_FOUND`. |
| PATCH | `/api/epics/:epicId` | `updateEpic` | Updates title/description only; 404 `EPIC_NOT_FOUND`. |
| DELETE | `/api/epics/:epicId` | `deleteEpic` | 204; 404 `EPIC_NOT_FOUND`; **409 `EPIC_REFERENCED` when referenced by tickets** (no cascade). |

## Acceptance criteria — verified
- **Epic team immutable after creation.** The `updateEpic` body schema is `additionalProperties: false` with no
  `teamId` property, so any request attempting to change the team is rejected with 400 `VALIDATION_ERROR` at the
  contract boundary. There is no code path in the module that writes `teamId` after `create`. Team is set once in
  `createEpic`. ✔
- **Title trim / non-empty enforced.** `normalizeTitle()` trims and throws 400 `VALIDATION_ERROR` on an
  empty-after-trim title, catching whitespace-only titles that pass the schema's `minLength: 1`. Applied on both
  create and update. ✔
- **Team-scoped list correct.** `listEpics` with `?teamId=X` filters `prisma.epic.findMany({ where: { teamId } })`,
  ordered by title; without the query it returns all epics, ordered by title. ✔
- **Delete-referenced-epic → 409.** `deleteEpic` counts `prisma.ticket.count({ where: { epicId } })`; if > 0 it
  throws `ConflictError('EPIC_REFERENCED', …)` (409). A FK-RESTRICT safety net (P2003/P2014, for a ticket inserted
  between the check and the delete) also maps to the same 409. No cascade. ✔
- **Timestamps server-set UTC.** `createdAt` (DB `@default(now())`) and `modifiedAt` (`@updatedAt`) are DB-managed;
  the API never accepts them from the client. Emitted as ISO-8601 UTC strings via `toEpicDTO`. ✔
- **Build clean:** `npm run build -w backend` passes (`tsc && tsc-alias`); frozen specifiers `backend/src/db`,
  `backend/src/core/errors` resolve at compile time and are rewritten to relative paths in `dist`. ✔

## Notes for other agents / flags to reconcile

1. **Error code for delete-referenced (RESOLVED at GATE-1).** The delete-referenced-epic 409 emits
   **`EPIC_REFERENCED`**, matching the FROZEN `contracts/openapi.yaml` (deleteEpic 409 example) and the
   `@app/shared` `errorCodes` registry. An earlier charter example used `EPIC_HAS_TICKETS`; the coordinator
   adjudicated the contract as authoritative (charter example corrected), so the module was changed to
   `EPIC_REFERENCED` (status 409 / message / `Error` envelope unchanged). **A11 (FE epic UI)** should key its 409
   handling off `EPIC_REFERENCED`.

2. **`listTeamEpics` (`GET /api/teams/{teamId}/epics`) is NOT in this module.** That path sits under the
   `/api/teams` prefix; a plugin mounted at `/api/epics` cannot register it (the prefix is applied by A4 and cannot
   be escaped). It therefore belongs to the **teams module (A6)** and is currently unimplemented there. The
   equivalent team-scoped listing consumers need is fully provided here as `GET /api/epics?teamId=X`. If the
   `/api/teams/{teamId}/epics` route is required, A6 should add it (or A1/MO reassign ownership).

3. **A4 wiring reminder.** `epicsModule` is delivered but not yet registered — A4 owns `backend/src/app.ts` /
   `backend/src/core/modules.ts` and must append `{ plugin: epicsModule, prefix: '/api/epics' }` (import from
   `backend/src/modules/epics`) to `featureModules`.

4. **Description handling.** Optional; stored as-is (`null` when omitted), preserving Markdown/whitespace. Trimming
   is applied only to title (per §5, which requires non-empty title but only an "optional description").
