# Handoff — A12 (Frontend Board & Drag-and-Drop), Phase 1

**Scope:** the Kanban board screen (REQUIREMENTS §8) — team selector, five state columns,
@dnd-kit drag-and-drop with optimistic persistence + rollback, and client-side filters/search.
Built on A9's foundation (ADR-0008 / ADR-0009) and navigates to tickets via A13's frozen
`ticketRoutes` (ADR-0008). Detail/create views are owned by A13, not implemented here.

## How it satisfies the contracts

- **Foundation (ADR-0008):** consumes only `@/lib/api` (`api`), `@/lib/auth` (`RequireAuth`), and
  `@/components` (`Toast`, `LoadingState`, `EmptyState`, `ErrorState`, `Button`, `Input`). No second
  API client or auth context is created. Routes are contributed via the `routes` export and picked up
  by A9's `import.meta.glob` auto-discovery — no edit to `@/app`.
- **Ticket navigation (ADR-0008, A13→A12):** `import { ticketRoutes } from '@/features/tickets/routes'`;
  `ticketRoutes.create()` for "New ticket", `ticketRoutes.detail(id)` on card click. No ticket URL is
  hard-coded. A13's file is present in-tree; the import resolves and typechecks.
- **API transport (ADR-0009):** all calls go through `@/lib/api` (`baseUrl: '/api'`). Endpoints used:
  `GET /teams`, `GET /teams/{teamId}/epics`, `GET /tickets?teamId=…`, `PATCH /tickets/{ticketId}`.

## Design decisions

- **DnD engine:** `@dnd-kit/core` `useDraggable` (cards) + `useDroppable` (columns). Within-column order
  is server/`modifiedAt`-driven with **no manually persisted order** (REQUIREMENTS §8), so cross-column
  moves are all that's needed — `@dnd-kit/sortable` reordering is intentionally not used. A `PointerSensor`
  activation distance (5px) lets a plain click open a ticket without starting a drag; a `KeyboardSensor`
  is registered for keyboard drag. A `DragOverlay` renders the lifted card for a smooth drag.
- **Client-side filtering:** the board fetches ALL tickets for the selected team (one cache entry:
  `['board','tickets',teamId]`) and applies type/epic/title filters client-side, memoized. This keeps
  filtering/search instant (no network churn on keystrokes), AND-combines cleanly, and makes the
  optimistic move update trivially correct. Cards are `React.memo`'d for smoothness at ≥100 tickets.
  (The server also supports `type`/`epicId`/`q` params per the contract if a future switch to
  server-side filtering is wanted.)
- **Optimistic move + rollback:** `useMoveTicket` `onMutate` snapshots the cache, moves the card to the
  new state and bumps `modifiedAt` (so it lands atop the destination column). `onError` restores the
  snapshot (card returns to its previous column) and BoardPage shows an error `Toast`. `onSuccess`
  reconciles the one ticket with the server's authoritative response. Persistence is via the `PATCH`,
  so a refresh (refetch) reflects the server state.
- **Team selection is URL-backed** (`/board?team=<id>`), defaulting to the first team, so the chosen
  board survives a refresh. Filters reset when switching teams.

## Notes / advisories for downstream & integration

- **Board discoverability / primary screen:** `@/app/router` (A9-owned) keeps a placeholder `Home` at
  index `/`; A12 can only add routes, so the board is registered at **`/board`**. There is currently no
  header nav link to it. If the board should be the app's landing screen, the shell/router owner (A9)
  should either redirect `/` → `/board` or add a nav link — that is outside A12's ownership.
- **Auth token on requests:** `@/lib/api` (A9) currently attaches no `Authorization` header. Against the
  Prism mock this is irrelevant; against the real backend, token attachment must be added by the
  foundation/auth owner (A9/A10) — the board just uses `api` as delivered and does not work around it.
- **Mock refetch churn:** on drag success the board keeps the optimistic result and reconciles the single
  ticket from the PATCH response (no blanket invalidate), so it stays stable under the Prism mock (which
  returns fresh random data on refetch). Real-backend correctness is unaffected.
- **No new dependencies** were introduced; only TDR-pinned `@dnd-kit/core` 6.3.1 and `@app/shared` /
  `@tanstack/react-query` / `react-router-dom` from the frozen stack are used.
