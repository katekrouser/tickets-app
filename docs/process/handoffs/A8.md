# A8 — Comments (Phase 1) Handoff

- **Owner:** A8 (Comments Engineer)
- **Scope:** REQUIREMENTS §7 — append-only ticket comments.
- **Files owned / delivered:** `backend/src/modules/comments/index.ts` (only file written).
- **Status:** Complete. `npm run build -w backend` clean; all acceptance criteria verified.

## What was built
Default-export `commentsModule: FastifyPluginAsync` (ADR-0005), mounted by A4 at prefix
`/api/tickets`, defining two routes:

| Method | Route (relative)        | operationId     | Behavior |
|--------|-------------------------|-----------------|----------|
| GET    | `/:ticketId/comments`   | `listComments`  | Comments for the ticket, chronological oldest-first. |
| POST   | `/:ticketId/comments`   | `createComment` | Adds a comment; 201 with the created `Comment`. |

Semantics:
- **Author** = `request.authUser.id` (global auth, ADR-0004); clients cannot set it.
- **created_at** is server-set UTC (Prisma `@default(now())`), serialized as ISO-8601 via
  `Date.toISOString()`.
- **Body non-empty**: enforced by `schemas.createComment` (Ajv `minLength: 1`) → empty body is a
  400 before the handler; a whitespace-only body is additionally rejected with
  `ValidationError('COMMENT_BODY_REQUIRED')` (400).
- **Ordering**: `orderBy: [{ createdAt: 'asc' }, { id: 'asc' }]` — oldest-first, with `id` as a
  stable tie-breaker for same-millisecond timestamps.
- **Comments are immutable** — no edit/delete routes (stretch only, out of scope).
- **Unknown ticket** → `NotFoundError('TICKET_NOT_FOUND')` (404) on both routes.

## Contracts honored
- **ADR-0002 (DB client):** `import { prisma } from 'backend/src/db'`; no local `PrismaClient`.
- **ADR-0003 (errors):** throws `NotFoundError` / `ValidationError` from `backend/src/core/errors`;
  never sets HTTP status for error paths manually.
- **ADR-0005 (module registration):** single default-export `FastifyPluginAsync`; no prefix
  hard-coded, no `app.listen`, no global hooks/decorators/auth inside the module.
- **ADR-0006 (`@app/shared`):** request/response validation via `schemas.listComments` /
  `schemas.createComment` — no hand-authored schemas.

## No-bump guarantee (§7)
Adding a comment inserts a `comment` row **only** — `prisma.comment.create` never writes the parent
`ticket` row, so the ticket's `modifiedAt` (`@updatedAt`) does not advance. This does not touch the
ticket-update path (A7's). Verified below.

## Coordination note for A4
Mount the module by appending to `backend/src/core/modules.ts` (A4-owned):
```ts

// ...

```
Register it **after** the tickets module so both live under `/api/tickets` (order is not functionally
significant; the paths do not collide).

## Verification (evidence)
- `npm run build -w backend` → clean (`tsc && tsc-alias`, no errors).
- Behavioral check via Fastify `inject` against the built plugin (stubbed `prisma` singleton, no live
  DB available in this environment). All PASS:
  - empty body → 400; whitespace-only body → 400
  - valid create → 201, `authorId` = authenticated user, `createdAt` ISO-8601 string

  - ticket `modifiedAt` unchanged after commenting
