## Endpoints + operationIds (23 operations, base path `/api`)

All business/auth routes are under the `/api` base path (expressed via the OpenAPI `servers` entry; individual
paths are root-relative so `openapi-fetch` with the frozen `baseUrl: '/api'` composes them correctly — ADR-0009 §4).

**Public** (no auth): `signup`, `login`, `verifyEmail`, `resendVerification`. Everything else requires a bearer JWT.

## `@app/shared` export surface (per ADR-0006)

Consumers import **only** via `@app/shared` (never a relative path).

```ts
// generated OpenAPI types
export type { paths, components, operations } from './openapi';

// enum constants + arrays + types (mirror the spec exactly)
export const TicketType  = { bug, feature, fix } as const;         export type TicketType;
export const TICKET_TYPES: readonly TicketType[];
export const TicketState = { new, ready_for_implementation, in_progress, ready_for_acceptance, done } as const;
export type TicketState;   export const TICKET_STATES: readonly TicketState[];

// domain error-code registry (ADR-0003 codes) + ErrorCode type
export const errorCodes;   export type ErrorCode;

// JSON-Schema validation API for A4's Fastify/Ajv
export interface RouteSchemas { body?; params?; querystring?; response: Record<number, JSONSchema7>; }
export const schemas: Record<string, RouteSchemas>;   // keyed by operationId
```

`schemas` is fully dereferenced (no `$ref`), so each entry is a self-contained Fastify `schema` object:
```ts
import { schemas } from '@app/shared';
app.post('/tickets', { schema: schemas['createTicket'] }, handler);
```

**Enum values are exact (REQUIREMENTS §6):** type `[bug, feature, fix]`; state `[new,
ready_for_implementation, in_progress, ready_for_acceptance, done]`.

## Production packaging (Phase 2 fix — prod-runtime blocker found by A5)

**Problem:** the package originally set `exports`/`types` to `./src/index.ts` and used extensionless internal
imports (`./schemas`, `./openapi`), so it only loaded under tsx/Vite — plain Node ESM (`node dist/server.js`)
could not resolve it.

**Fix (packaging only — ADR-0006 export surface unchanged and still frozen):**
- Internal relative imports now use explicit `.js` specifiers (NodeNext): `index.ts` → `./schemas.js`,
  `./openapi.js`; generated `schemas.ts` → `./index.js` (the generator emits this).
- Added `npm run build` = `node ./scripts/build.mjs`, which runs `tsc -p tsconfig.build.json`
  (`module`/`moduleResolution: NodeNext`, `declaration: true`, `verbatimModuleSyntax`) to emit
  `dist/index.js`, `dist/schemas.js` and their `.d.ts`, then copies `src/openapi.d.ts` → `dist/openapi.d.ts`
  (tsc does not emit `.d.ts` inputs; the OpenAPI types are declaration-only, so the value bundle never imports
  them at runtime, but the built type surface `paths/components/operations` must still resolve — ADR-0006).
- `package.json` now points at the build:
  `main`/`module` → `dist/index.js`; `types` → `dist/index.d.ts`;
  `exports["."]` = `{ "types": "./dist/index.d.ts", "import": "./dist/index.js" }`; `files` = `["dist","src"]`.
- `src/**` kept intact. Added devDependency **`typescript` ^5.6.0** (matches the TDR TypeScript pin) for the build.
- **Note for A2/A4 (Dockerfile owners):** the backend image's `npm run build --workspace @app/shared` step now
  produces `dist` (previously a no-op). It requires devDependencies (`typescript`) to be present at build time —
  install deps *before* `--omit=dev` pruning (multi-stage: install all → build shared → prune). This is a
  Dockerfile concern (outside `packages/shared/**`); flagged here so it is not missed.

**Verified:**
- `npm run build -w @app/shared` produces `dist/` with `index.js` (enums + `errorCodes` + `export { schemas } from './schemas.js'`, the type-only `./openapi.js` export fully erased), `schemas.js`, and all `.d.ts`.
- Node ESM consumer (`import('@app/shared')` resolved via the package `exports` map, i.e. exactly what
  `node dist/server.js` does) loads `schemas` (23 ops), `TicketType`, `TICKET_STATES`, `errorCodes` **without error**.
- NodeNext TypeScript consumer resolves `paths`/`components`/`operations`, `RouteSchemas`, `schemas`, and the
  enum types from `dist` — `tsc --noEmit` exit 0.

Build-only devDependency added for the schema extractor: **`js-yaml` ^4.1.0** (parses the YAML spec; not shipped
at runtime, does not affect the frozen runtime stack). Also devDeps: `openapi-typescript ^7`, `@stoplight/prism-cli ^5`;
runtime dep: `@types/json-schema` (types only). All MIT-licensed.

## Note for A9 (frontend) — Prism base-path behavior (advisory, not a blocker)

Prism 5 does **not** apply the OpenAPI `servers` base path (`/api`) to mocked routes — it serves operations at the
**root** (e.g. `GET http://localhost:3000/teams`, not `/api/teams`). This is a documented Prism limitation and was
confirmed by direct test (even a host-qualified `http://localhost:3000/api` server was ignored). The real backend
serves under `/api/*` (ADR-0009 §5/§6). Consequence: the mock and the real backend are **not** byte-identical behind
one unmodified `/api` Vite proxy. When developing against the Prism mock, configure the dev proxy to strip the
`/api` prefix (`rewrite: p => p.replace(/^\/api/, '')`); when pointing at the real backend, keep the path preserved.
The path keys in the generated types remain root-relative, which is exactly what `openapi-fetch` (`baseUrl: '/api'`)
expects, so no type/client change is needed. Changing the contract to bake `/api` into every path was rejected
because it would double-prefix `openapi-fetch` requests to `/api/api/...`.
