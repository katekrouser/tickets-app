# Requirements

The canonical scope of the project — the single source of truth for *what* to build.

| File | Owner | Contents |
|---|---|---|
| [REQUIREMENTS.md](REQUIREMENTS.md) | fixed (extracted from `Hackathon Task.docx`) | Objective, architecture, auth, teams, epics, tickets, comments, board, API/persistence, screens, non-functional, out-of-scope, and the §13 Definition of Done |

Every agent reads this first. It is not modified during the build; if scope questions arise they are escalated to
the Master Orchestrator, and decisions are recorded as ADRs / in the [Decision Log](../governance/decision-log.md).
Verification that every clause is met is tracked in the QA **Requirement Traceability Matrix** (`../quality/qa/rtm.md`).
